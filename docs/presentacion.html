💡 ¿Qué es Codex?

Codex es una plataforma educativa web diseñada para enseñar desarrollo frontend de forma estructurada e interactiva. Los alumnos aprenden HTML, CSS y JavaScript a través de lecciones con teoría, ejemplos de código y quizzes de evaluación.

El profesor gestiona el contenido, supervisa el progreso de los alumnos en tiempo real y puede añadir nuevos estudiantes desde un panel de administración dedicado.

🎓 Para alumnos
Lecciones interactivas con código, quizzes de evaluación, seguimiento de progreso y sistema de logros.
👨‍🏫 Para profesores
Panel de control con estadísticas globales, gestión de alumnos, lecciones y valoraciones.

JWT Payload
{
  "sub": "userId",
  "email": "user@codex.com",
  "rol": "alumno" | "profesor",
  "iat": 1718000000,
  "exp": 1718604800 // 7 días
}
  • Contraseñas hasheadas con bcrypt (salt rounds: 10)
  • Tokens JWT firmados con secreto de entorno (JWT_SECRET)
  • Guards aplicados globalmente en todos los endpoints protegidos
  • Control de rol por endpoint: acciones de profesor verificadas en servidor
  • Interceptor Angular para adjuntar token y redirigir en 401
  • Fallback por email en JWT strategy para tolerancia a re-seeds
  • Separación de rutas Angular por rol — alumnos no pueden acceder a /profesor/*
  • Variables de entorno en .env — credenciales fuera del código fuente

🧩 Retos técnicos superados
CORS y proxy de desarrollo
Las peticiones directas al backend desde el navegador quedaban colgadas. Solución: proxy de Angular que redirige /api/* al servidor NestJS y prefijo global /api en el backend.
Change detection en Angular 21
Las respuestas HTTP completaban correctamente pero la vista no se actualizaba. Causa: zone.js no parchea el Fetch API de Vite. Solución: ChangeDetectorRef.detectChanges() en cada callback.
Caché HTTP 304 Not Modified
Express genera ETags automáticamente. El navegador enviaba If-None-Match y recibía 304 sin cuerpo, dejando Angular en estado de carga infinita. Solución: app.set('etag', false).
DNS intermitente con MongoDB Atlas
Resolución DNS fallida para los hostnames de shard en macOS. Causa: Node.js intentaba resolver via IPv6. Solución: family: 4 en las opciones de Mongoose para forzar IPv4.
Race condition en interceptor JWT
Tres peticiones paralelas fallando con 401 causaban múltiples llamadas a localStorage.clear() y navigate('/login'). Solución: flag redirectingToLogin a nivel de módulo.
Invalidación de tokens tras re-seed
Re-ejecutar el seed borra y recrea los usuarios con nuevos IDs. Los tokens antiguos apuntaban a IDs inexistentes. Solución: fallback por email en la JWT Strategy de Passport.
